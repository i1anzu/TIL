##  HTTP Method

HTTP 메서드는 클라이언트가 서버에게 특정 리소스에 대해 **수행하고자 하는 동작**을 정의합니다.

### 주요 메서드 4가지


| 메서드 | 역할 (CRUD) | 설명 |
| :--- | :--- | :--- |
| **GET** | **Read** (조회) | 리소스를 조회합니다. URL의 쿼리 스트링을 통해 데이터를 전달합니다. |
| **POST** | **Create** (생성) | 서버에 데이터를 전달하여 새로운 리소스를 생성합니다. 메시지 Body를 사용합니다. |
| **PUT** | **Update** (수정) | 리소스를 완전히 대체합니다. 해당 리소스가 없으면 새로 생성하기도 합니다. |
| **DELETE** | **Delete** (삭제) | 특정 리소스를 삭제합니다. |

---

### 핵심 차이점

#### 안전(Safe)과 멱등성(Idempotent)
*   **안전(Safe):** 호출해도 리소스가 변경되지 않는 성질 (**GET**)
*   **멱등성(Idempotent):** 동일한 요청을 여러 번 보내도 결과가 똑같은 성질 (**GET, PUT, DELETE**)
    *   *POST는 호출할 때마다 새로운 리소스가 생길 수 있으므로 멱등하지 않습니다.*

#### 데이터 전달 방식
*   **GET:** URL 경로 뒤에 `?name=value` 형태로 데이터를 붙여 보냅니다. (보안에 취약, 캐싱 가능)
*   **POST/PUT:** HTTP 메시지의 **Body**에 데이터를 담아 보냅니다. (대용량 데이터 및 보안이 필요한 정보에 적합)

---

### 요약
- **조회**할 때는 `GET`
- **등록**할 때는 `POST`
- **수정**할 때는 `PUT` (또는 부분 수정 시 `PATCH`)
- **삭제**할 때는 `DELETE`
