### HTTP Status Code

서버가 클라이언트의 요청에 대해 어떤 처리 결과를 내놓았는지 알려주는 3자리 숫자입니다.

*   **`200` (OK):** 요청이 **성공**적으로 처리됨
*   **`201` (Created):** 요청이 성공하여 새로운 리소스가 **생성**됨 (주로 POST 요청 후 응답)
*   **`400` (Bad Request):** 클라이언트의 요청이 잘못되어 서버가 이해할 수 없음 (**잘못된 요청**)
*   **`401`, `403` (Unauthorized / Forbidden):** 해당 리소스에 접근하기 위한 **인증**이 필요하거나 **권한**이 없음
*   **`404` (Not Found):** 요청한 리소스를 **찾을 수 없음** (주로 URL이 틀렸을 때 발생)
*   **`500` (Internal Server Error):** 서버 내부 문제로 인해 에러가 발생함 (**서버 오류**)

## HTTP Status Code 상세 정리

상태 코드는 백엔드와 프론트엔드가 소통하는 가장 기본적인 약속입니다. 앞자리에 따라 대략적인 상태를 알 수 있습니다.

### 2xx (Successful): 요청을 성공적으로 처리함
*   **`200` (OK):** 가장 흔한 성공 응답. 데이터를 성공적으로 불러왔을 때 사용합니다.
*   **`201` (Created):** 요청이 성공하여 새로운 리소스(게시글, 계정 등)가 생성되었을 때 사용합니다.
*   **`204` (No Content):** 요청은 성공했지만, 응답 Body에 보낼 데이터는 없을 때 사용합니다. (예: 삭제 성공)

### 3xx (Redirection): 요청을 마치기 위해 추가 동작이 필요함
*   **`301` (Moved Permanently):** 요청한 리소스의 URL이 영구적으로 변경되었음을 알립니다.
*   **`302` (Found):** 일시적인 이동을 의미하며, 임시적으로 다른 페이지로 보낼 때 사용합니다.

### 4xx (Client Error): 클라이언트(나)의 요청에 문제가 있음
*   **`400` (Bad Request):** 서버가 요청을 이해하지 못함. (파라미터가 빠졌거나 오타가 난 경우)
*   **`401` (Unauthorized):** 비로그인 상태이거나 인증 정보가 잘못되어 누구인지 알 수 없을 때 발생합니다.
*   **`403` (Forbidden):** 로그인은 되어있으나, 해당 리소스에 접근할 권한이 없을 때 발생합니다. (예: 일반 유저가 관리자 페이지 접속)
*   **`404` (Not Found):** 요청한 주소에 리소스가 없음. 가장 흔히 보는 에러 페이지입니다.

### 5xx (Server Error): 서버(상대방)가 문제를 일으킴
*   **`500` (Internal Server Error):** 서버 내부 로직(코드)에 에러가 발생한 경우입니다. (가장 흔한 서버 에러)
*   **`502` (Bad Gateway):** 서버와 서버 사이의 통신 중 문제가 발생했을 때 나타납니다.
*   **`503` (Service Unavailable):** 서버가 과부하 걸렸거나 점검 중이라 일시적으로 사용할 수 없는 상태입니다.
