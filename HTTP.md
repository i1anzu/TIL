# HTTP
**HTTP (HyperText Transfer Protocol)**: 인터넷에서 데이터를 주고받기 위한 기본 약속입니다. 하지만 데이터를 암호화하지 않고 생으로 보내기 때문에 중간에 누가 훔쳐볼 위험이 있습니다.

**HTTPS (HTTP Secure)**: HTTP에 보안(SSL/TLS 암호화) 레이어를 추가한 버전입니다. 데이터가 암호화되어 전송되므로 해킹으로부터 훨씬 안전합니다. 

**HTTPS가 꼭 필요한 이유**:
- **기밀성**: 로그인 정보나 결제 데이터 같은 민감한 정보를 암호화하여 보호합니다.
- **무결성**: 데이터가 전송되는 도중에 변조되지 않았음을 보장합니다.
- **신뢰성**: 접속한 사이트가 진짜 그 사이트가 맞는지 증명해 줍니다. (구글 크롬 등에서 '안전하지 않음' 경고가 뜨는 걸 방지합니다.) 

**요청(Request)과 응답(Response) 구조** : 웹은 클라이언트(나)가 요청을 보내면 서버가 응답을 주는 구조로 돌아갑니다.

### **Request (요청)**
클라이언트가 서버에 "이것 좀 해줘!"라고 보내는 메시지입니다.
- **Start Line**: 어떤 방식으로(GET, POST 등), 어디에(/main), 어떤 버전(HTTP/1.1)으로 요청할지 적습니다.
- **Header**: 보내는 데이터의 타입, 브라우저 정보, 쿠키 등 부가 정보를 담습니다.
- **Body**: 서버로 보낼 실제 데이터(로그인 ID/PW, 게시글 내용 등)가 들어갑니다. (GET 방식은 보통 비어있습니다.)
### **Response (응답)**
요청을 받은 서버가 결과를 돌려주는 메시지입니다.
- **Status Line**: 처리가 잘 됐는지 알려주는 상태 코드가 핵심입니다. (예: 성공은 200, 페이지 없음은 404)
- **Header**: 응답 데이터의 타입, 서버 정보, 캐시 정책 등을 담습니다.
- **Body**: 서버가 보내주는 실제 데이터(HTML 코드, 이미지 파일, JSON 데이터 등)가 들어갑니다.
