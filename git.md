# Git

**Git이란?** 소프트웨어 개발 시 소스 코드의 변경 사항을 기록하고 관리하는 분산 버전 관리 시스템(VCS)입니다. 여러 명의 개발자가 동시에 협업할 때 발생하는 코드 충돌을 방지하고, 프로젝트의 과거 기록을 보관하여 필요시 특정 시점으로 되돌릴 수 있게 해줍니다. 

<**핵심 개념**>
- **저장소 (Repository)**: 프로젝트의 모든 파일과 변경 이력이 저장되는 공간입니다. 내 컴퓨터에 있는 로컬 저장소와 서버에 위치한 원격 저장소로 나뉩니다.
- **커밋 (Commit)**: 변경된 사항을 저장소에 기록하는 '스냅샷'입니다. 의미 있는 변화가 생길 때마다 커밋을 하여 프로젝트의 역사를 기록합니다.
- **브랜치 (Branch)**: 원본 코드에서 뻗어 나온 '가지'입니다. 새로운 기능을 개발하거나 버그를 수정할 때 독립된 작업 공간을 만들어 안전하게 작업한 후, 나중에 원본에 합칠 수 있습니다.
- **체크아웃 (Checkout)**: 특정 브랜치로 이동하거나, 과거의 특정 커밋 시점으로 소스 코드를 되돌리는 동작입니다. 

<**작업 흐름 (Workflow)**>
- **Working Directory**: 현재 파일을 수정하고 있는 실제 작업 공간입니다.
- **Staging Area**: 커밋하기 전, 저장소에 올릴 파일들을 선별하여 대기시키는 장소입니다.
- **Git Directory (Repository)**: commit 명령을 통해 변경 사항이 영구적으로 저장되는 로컬 저장소입니다.

<**Git 명령어 개념**>
- **git init**: 새 Git 저장소를 초기화할 때 사용!
- **git clone [URL]**: 원격 저장소 복제해서 로컬에 가져오기
- **git status**: 현재 변경된 파일 상태 확인
- **git add [파일명]**: 변경된 파일을 스테이지에 올리기
- **git commit -m "메시지"**: 스테이지에 올라온 변경사항을 커밋하여 확정
- **git push**: 로컬 커밋 내용을 Github 같은 원격 저장소에 업로드
- **git pull**: 원격 저장소 변경사항을 가져와 병합
- **git log**: 커밋 히스토리 확인
- **git branch [브랜치 명]**: 현재 브랜치 기준으로 새로운 브랜치 생성
- **git checkout [브랜치명]**: 다른 브랜치로 이동
- **git stash**: 작업 중인 브랜치에서 변경된 파일을 임시 저장소에 보관
- **git stash pop**: 가장 최근에 저장한 스태시 워킹 디렉토리에 되돌려 적용, 해당 스태시 스택에서 제거
- **git checkout <파일명>**: 특정 커밋(또는 HEAD)의 상태로 개별 파일을 복원
