# GitHub

**Github란?** 전 세계 개발자들이 코드를 올리고, 서로 공유하며, 함께 소프트웨어를 만드는 웹 기반 소스 코드 저장소입니다.

<**주요 특징 및 기능**>
- **버전 관리 (Git 기반)**: 소스 코드의 수정 이력을 기록하는 도구인 'Git'을 클라우드에서 관리할 수 있게 해줍니다. 실수를 해도 이전 상태로 언제든 되돌릴 수 있습니다.
- **협업의 중심지**: 여러 사람이 동시에 하나의 프로젝트를 작업할 때, 코드가 꼬이지 않게 합치거나 리뷰하는 기능을 제공합니다.
- **오픈 소스 공유**: 전 세계 유명한 소프트웨어(예: 리눅스, 테슬라 자율주행 코드 일부 등)의 소스가 이곳에 공개되어 있어 누구나 보고 배울 수 있습니다.
- **개발자 포트폴리오**: 개발자가 어떤 코드를 짰고 얼마나 꾸준히 활동했는지 보여주는 지표가 되어, 취업 시 이력서처럼 활용되기도 합니다.

<**Branch(브랜치) 생성 및 사용하기**>
원본 코드(main)를 건드리지 않고 독립적으로 작업하기 위해 사용합니다.
- **생성 및 이동**: 터미널에서 git checkout -b 브랜치명 명령어를 입력하면 새 브랜치를 만들고 바로 이동합니다.
- **작업 및 푸시**: 코드 수정 후 다음 명령어를 순서대로 입력하여 GitHub에 올립니다.
1. git add . (변경 사항 추가)
2. git commit -m "메시지" (변경 사항 기록)
3. git push origin 브랜치명 (원격 저장소로 업로드)

<**Pull Request(PR) 생성하기**>
작업한 브랜치를 메인 브랜치에 합쳐달라고 요청하는 단계입니다.
- **생성 방법**: GitHub 저장소로 이동하면 보이는 Compare & pull request 노란색 버튼을 클릭합니다.
- **설정**: 합칠 기준 브랜치(base)와 내 작업 브랜치(compare)를 확인하고, 제목과 설명을 적은 뒤 Create pull request를 누릅니다.
- **연결**: 설명란에 fixed #이슈번호라고 적으면 해당 이슈와 자동으로 연결됩니다.

<**코드 리뷰 반영하기**>
동료의 피드백을 확인하고 코드를 수정하는 과정입니다.
- **피드백 확인**: PR 페이지의 Files changed 탭에서 팀원이 남긴 코멘트를 확인합니다.
- **수정 반영**:
1. 로컬(내 컴퓨터)의 동일한 브랜치에서 코드를 수정합니다.
2. 다시 add, commit, push 과정을 거치면 기존 PR에 변경 사항이 자동으로 업데이트됩니다
- **마무리**: : 리뷰어가 승인(Approve)하면 Merge pull request 버튼을 눌러 메인 브랜치에 합칩니다.

<**Issue(이슈) 생성하기**>
작업할 내용이나 버그를 기록하는 첫 단계입니다.
- **생성 방법**:  GitHub 저장소 상단의 Issues 탭 → New issue 클릭 → 제목과 내용을 입력 후 Submit new issue를 누릅니다.
- **활용**: 작업 담당자(Assignees)를 지정하거나 라벨(Labels)을 붙여 관리할 수 있습니다.
