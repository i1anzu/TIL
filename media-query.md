# media query

**반응형(media query)란?** 현대 웹 개발에서 기기에 상관없이 최적의 화면을 보여주기 위해 사용하는 핵심 기술입니다.

<**반응형 웹 디자인**>

사용자가 접속하는 기기의 화면 크기(해상도)에 맞춰 웹사이트의 레이아웃, 이미지 크기, 텍스트 등이 자동으로 변하는 방식입니다.
- **하나의 소스** : PC용, 모바일용 사이트를 따로 만들지 않고 하나의 코드(HTML)로 모든 기기에 대응합니다.
- **유동적인 레이아웃** : 화면 너비가 좁아지면 가로로 나열된 메뉴가 세로로 쌓이거나, 중요하지 않은 요소는 숨겨지는 등 유연하게 반응합니다.

<**핵심 개념**>
- **@media** : 반응형 스타일을 시작하겠다는 신호탄 입니다. 이 뒤에 조건을 붙여서 사용합니다.
- **max-width(최대 너비)** : "화면이 최대 이 너비까지 지만 이 스타일을 적용해줘"라는 뜻입니다. 보통 모바일기기를 타겟으로 할 때 많이 씁니다. (max-width: 768px 이렇게 코드를 쓰면 768px 이하인 화면에서 작동)
- **min-width(최소 너비)** : "화면이 적어도 이 너비 이상일 때 스타일을 적용해줘"라는 뜻입니다. 보통 PC나 큰 화면을 타겟으로 할 때 씁니다. (min-width: 1024px 이렇게 코드를 쓰면 1024px 이상인 화면에서 작동)

<**반응형 단위(Responsive Units)**>

고정된 크기(px) 대신, 주변 환경에 따라 크기가 변하는 가변적인 단위들입니다.
- **%** : 부모 요소 기준, 부모 박스 크기의 몇 퍼센트를 차지할지 결정합니다.
- **vw** : 뷰포트 너비 기준, 화면 가로 전체를 100으로 보고 그 비중을 정합니다. (100vw = 전체 너비)
- **vh** : 뷰포트 높이 기준, 화면 세로 전체를 100으로 보고 그 비중을 정합니다. (100vh = 전체 높이)
- **rem** : 최상위(뿌리) 글자 기준, 보통 브라우저 기본 폰트 크기(16px)를 기준으로 배수를 정합니다. (1rem = 16px)
