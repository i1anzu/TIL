# media query

**반응형(media query)란?** 현대 웹 개발에서 화면의 크기(너비, 높이)나 장치의 종류에 따라 서로 다른 스타일을 적용할 수 있게 해주는 기능입니다.

<**반응형 웹 디자인**>

사용자가 접속하는 기기의 화면 크기(해상도)에 맞춰 웹사이트의 레이아웃, 이미지 크기, 텍스트 등이 자동으로 변하는 방식입니다.
- **HTML** : 웹사이트의 뼈대와 내용을 만듭니다. (내용은 하나)
- **CSS 미디어 쿼리** : "화면 너비가 768px보다 작으면 모바일용 레이아웃을 보여주고, 그보다 크면 PC용 레이아웃을 보여줘"라고 지시하는 역할을 합니다.

HTML 하나만 있다고 해서 저절로 반응형 웹이 되지는 않고, **미디어 쿼리**가 기기 크기에 맞춰 이미지 크기를 줄이거나, 가로로 배치된 메뉴를 세로로 쌓는 등의 실제적인 변화를 만들어냅니다.

- **유동적인 레이아웃** : 화면 너비가 좁아지면 가로로 나열된 메뉴가 세로로 쌓이거나, 중요하지 않은 요소는 숨겨지는 등 유연하게 반응합니다.

**and 연산자**: "둘 다 맞아야 해"
가장 많이 쓰이고, 특정 범위(최소~최대)를 지정할 때 유용합니다.
```
@media (min-width: 768px) and (max-width: 1024px) {}
```
**, (쉼표, or 역할)**: "둘 중 하나라도 맞으면 돼"
CSS 미디어 쿼리에서는 or 대신 쉼표(,)를 사용합니다.
```
@media (orientation: landscape), (min-width: 1200px) {}
```
**not 연산자**: "이 조건만 아니면 돼"
전체 조건을 반전시킵니다.
```
@media not print {}
```

<**핵심 개념**>
- **@media** : 반응형 스타일을 시작하겠다는 신호탄 입니다. 이 뒤에 조건을 붙여서 사용합니다.
- **max-width(최대 너비)** : "화면이 최대 이 너비까지 지만 이 스타일을 적용해줘"라는 뜻입니다. 보통 모바일기기를 타겟으로 할 때 많이 씁니다. (max-width: 768px 이렇게 코드를 쓰면 768px 이하인 화면에서 작동)
- **min-width(최소 너비)** : "화면이 적어도 이 너비 이상일 때 스타일을 적용해줘"라는 뜻입니다. 보통 PC나 큰 화면을 타겟으로 할 때 씁니다. (min-width: 1024px 이렇게 코드를 쓰면 1024px 이상인 화면에서 작동)

<**반응형 단위(Responsive Units)**>

고정된 크기(px) 대신, 주변 환경에 따라 크기가 변하는 가변적인 단위들입니다.
- **%** : 부모 요소 기준, 부모 박스 크기의 몇 퍼센트를 차지할지 결정합니다.
- **vw** : 뷰포트 너비 기준, 화면 가로 전체를 100으로 보고 그 비중을 정합니다. (100vw = 전체 너비)
- **vh** : 뷰포트 높이 기준, 화면 세로 전체를 100으로 보고 그 비중을 정합니다. (100vh = 전체 높이)
- **rem** : 최상위(뿌리) 글자 기준, 보통 브라우저 기본 폰트 크기(16px)를 기준으로 배수를 정합니다. (1rem = 16px)

<**Flex(1차원 레이아웃)**>

요소들을 한 줄(가로 또는 세로)로 정렬할 때 최적화되어 있습니다.
- **display: flex** : 부모 박스에 선언하여 "이제부터 이 안의 자식들을 유연하게 배치하겠다"고 선언하는 시작점입니다.
- **justify-content** : 메인축(주로 가로) 정렬입니다. (왼쪽/가운데/오른쪽 정렬, 요소 사이 간격 동일하게 배치 등)
- **align-items** : 수직축(주로 세로) 정렬입니다. (위/가운데/아래 정렬)

<**Grid(2차원 레이아웃)**>

가로(행)와 세로(열)을 동시에 조절하여 복잡한 격자무늬 레이아웃을 만들 때 사용합니다.
- **display: grid** : 그리드 레이아웃을 시작합니다.
- **grid-template-columns** : 가로줄(열)의 개수와 너비를 정합니다. (1fr 1fr 1fr은 3등분)
- **grid-template-rows** : 세로줄(행)의 높이를 정합니다.

<**Flex와 Grid의 차이점**>

- **핵심** : Flex는 1차원으로 한 방향(가로 혹은 세로)로 배치되는 반면 Grid는 2차원으로 가로와 세로를 동시에 배치한다.
- **용도** : Flex는 메뉴 바, 리스트 아이템 정렬 / Grid는 복잡한 메인 페이지 레이아웃, 대시보드에 사용된다.

Flex는 기차 칸처럼 한 줄로 연결, Grid는 바둑판이나 달력처럼 칸 나누기라고 생각하면 된다.
